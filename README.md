# Marble Marvelous

A 3D "Blue Marble" Earth live wallpaper for Android, rendered with libGDX + GLSL
shaders: day/night maps, atmosphere, specular ocean, city lights at the
terminator, and a live cloud layer fetched from OpenWeather.

## Status
- ✅ `./gradlew assembleDebug` builds a working `app-debug.apk`.
- ✅ Opens in Android Studio; Gradle sync works.
- ✅ **Runtime-verified** on an emulator: the Earth wallpaper renders (globe,
  clouds, continents, city lights, terminator), touch works, no crash.
- Written in **Kotlin** (69 Kotlin / 1 hand-written Java file) on **AndroidX**.

## Requirements
- **JDK 21** to run Gradle. The repo pins it via `org.gradle.java.home` in
  `gradle.properties` (currently a Homebrew path,
  `/opt/homebrew/opt/openjdk@21/...`) — change that line to your JDK 21 path, or
  remove it if your default JDK is already 21.
- Android SDK **API 36** installed (compileSdk = 36).

## Open / build
1. Android Studio → **File ▸ Open** → this folder, let Gradle sync.
2. Or CLI: `./gradlew assembleDebug` → `app/build/outputs/apk/debug/app-debug.apk`.

Set it as a live wallpaper (headless):
```sh
adb shell am start -a android.service.wallpaper.CHANGE_LIVE_WALLPAPER \
  -n com.android.wallpaper.livepicker/.LiveWallpaperChange \
  --ecn android.service.wallpaper.extra.LIVE_WALLPAPER_COMPONENT \
  com.phuchienngo.marblemarvelous/.celestialBodies.wallpapers.EarthWallpaperService
```
then tap **Set wallpaper**.

## Architecture
- **Render**: libGDX engine + GLSL shaders (`app/src/main/assets/earth/*.{vert,frag}`,
  shared `assets/glsl/*`).
- **DI**: Dagger 2 with `@Inject` constructors, generated by KSP. Two components —
  `WallpaperComponent` (lifecycle controllers) and `EngineComponent` (weather
  providers).
- **Concurrency**: structured coroutines; weather fetches run off the GL thread.

## Weather / clouds
The cloud layer is fetched live from **OpenWeatherMap**:
- `weather/OpenWeatherClouds.kt` downloads `clouds_new` web-mercator tiles and
  **re-projects them onto the 6 OpenGL cube faces** (`{px,nx,py,ny,pz,nz}.png` in
  `cacheDir`).
- The source world is held as a **1-byte-per-pixel grayscale field** (not an ARGB
  bitmap), so a high-resolution cloud source fits in memory: `SRC_ZOOM = 5`
  (8192px / 1024 tiles) -> ~2048px per equatorial cube face, matching the 2048²
  earth maps. Tiles are streamed and recycled (peak ~114 MB); downloads retry on
  transient failures. A full refresh is **download-bound** (~1024 sequential
  tiles), runs in the background, and is cached (re-fetched only when older than
  3 h). Lower `SRC_ZOOM` to 4 for a faster (~256-tile) but softer refresh.
- Generated cloud faces are lightly smoothed before being written to cache, and
  the Earth shader samples clouds in tangent space so the live layer keeps cloud
  detail without obvious cubemap-axis streaks.
- **API key**: set `openweather_api_key` in `app/src/main/res/values/strings.xml`
  (note: it is embedded in the APK). Empty key → the bundled `earth/clouds.ktx`.
- Lightning **storms** (`StormsProvider`) point at an endpoint that is currently
  unavailable and fail gracefully (the globe still renders).

## Native libraries (`app/src/main/jniLibs`)
Only **`libgdx.so`** is bundled (one per ABI: `arm64-v8a`, `armeabi-v7a`, `x86`,
`x86_64`), loaded by the libGDX backend.

## Notes
The single hand-written Java file is intentional:
`celestialBodies/shaders/attributes/PlanetTextureAttribute.java` extends a libGDX
`Attribute` using a protected-static `register()`, which is awkward in Kotlin.
Generated protobuf-lite code lives under `app/src/main/generated/java` as
`weather/StormProtos.java`.
